# Paper Reading Notes

辅助论文阅读和笔记整理。支持在对话中逐章节记录笔记，读完后整理成结构化文档。

## Usage

```
/paper-reading <论文名称>              # 创建/打开论文笔记
/paper-reading tidy <论文名称>         # 整理已有笔记
```

## Instructions

### 创建笔记

当用户提供论文名称或 PDF 路径时：

1. 在 `paper-reading/` 目录创建笔记文件，文件名使用论文简称（如 `FastUMI-100K.md`）
2. 使用以下最小模板，不预设章节结构：

```markdown
# <论文简称> 论文笔记

## 论文信息

- **标题**: <完整标题>
- **链接**: <arXiv 或其他链接>

---

<!-- 以下内容在阅读过程中逐步添加 -->
```

3. 如果 `paper-reading/` 中已有同名文件，直接打开而非覆盖

### 阅读过程中

用户会粘贴论文段落并可能附带评论，按以下规则处理：

1. **论文内容** → 翻译整理为中文笔记，保留关键术语英文原文，写入对应章节
2. **用户评论**（以 "notes："、"💭"、"评价："等标记）→ 以引用格式 `>` 记录
3. **章节标题**由论文内容动态决定，不提前预设
4. 保留论文中的公式、表格、代码块等结构化内容
5. 图表引用使用 `(Fig.N)` 格式

### 整理笔记 (`tidy`)

读完后执行整理：

1. **删除空白章节** — 移除仅包含 `[待补充]` 的章节
2. **按阅读顺序重组** — 将章节排列为用户实际阅读的顺序，而非论文原始顺序
3. **汇总个人笔记** — 将分散在各处的引用格式评价（`>` 开头的个人评论）收集到末尾统一的「个人笔记」部分，并去掉引用格式
4. **标题简化** — 将「论文精读」改为「论文笔记」（除非用户确实精读了全文）

## Examples

```
/paper-reading FastUMI-100K
# 创建 paper-reading/FastUMI-100K.md 并初始化模板

/paper-reading tidy FastUMI-100K
# 整理 paper-reading/FastUMI-100K.md：删除空白章节、重组、汇总笔记
```

## Notes

- 笔记语言为中文，关键术语保留英文
- 不添加用户未阅读的内容（如摘要、相关工作等空白章节）
- 个人评论和论文内容明确区分
